# Temporal One — концепция

> Living doc. Решённое перемещаем сверху, открытые вопросы держим явными.
> Последнее обновление: 2026-04-25.

---

## Идея в одном абзаце

Платформа для безработных в IT. Они собираются в команды по 5 человек, месяц делают стартап на минимальный бюджет (~$200 на всю команду — Claude, трафик), ведут **верифицированный лог работы** — и либо запускают бизнес, либо получают доказательство своих навыков, которое продаётся нанимателям. Работодатели платят за доступ к логам.

Аналог Stripe-верификации из TrustMRR — только верифицируется не выручка, а реальный трудовой вклад.

---

## Контекст рынка (для лендинга)

- 245 000+ уволенных в IT в 2025 (≈674 человека в день)
- Средний поиск работы — 6+ месяцев
- 61% соискателей получают ghosting после собеседований
- 18–22% вакансий — ghost jobs (не существуют)
- Средняя вакансия получает 242 заявки (×3 vs 2017)
- 41% резюме не видит ни один живой человек

**Главный диагноз:** сломан не «мало вакансий», а сам процесс найма. Огромные воронки, нулевая прозрачность, собеседования проверяют слова, а не работу.

## Три аудитории

**Builders (основные пользователи).** Не сидят без дела — реально работают и строят. Шанс запустить бизнес и не искать работу. Если не получилось — есть лог, продающий лучше резюме. Знакомятся с людьми: схантили одного — он может вытащить остальных.

**Leaders (отдельная роль).** Бывшие тимлиды, CTO, продакты, основатели — их режут первыми. Сами набирают команду из пула builder'ов. Получают статус на платформе, продаются дороже. Большая доля если стартап взлетит. Получают playbook — чёткий план с первого дня.

**Employers (платят).** Покупают доступ к базе логов. Видят как человек реально работает — не резюме, не театр собеседования. Открытый доступ — без верификации тиммейтов. Платный — полные данные + peer-верификация.

**На форме заявки — один toggle**: «I want to build» (включает leaders) vs «I'm hiring». Builder vs Leader выясняется внутри платформы.

## Verifiable work log — главная инновация

### Формат записи

Каждая запись содержит **5 обязательных полей**:

- *What I did* — одна строка
- *Artifact* — ссылка / файл / embed. **Без артефакта запись не существует**
- *Time spent* — часы
- *Self-tag* — built / fixed / researched / designed / shipped / blocked
- *Optional note* — что узнал, на чём затупил

И **2 скора от тиммейтов** в конце дня:

- *Skill* — «работа реальная и качественная» / «соврал или халтура»
- *Behavior* — «работал нормально» / «был сложным в чате»

Скоры анонимны для оценщика, видны только админу и нанимателю в платной части. Опционально — приватный комментарий («он наврал про PR #42»).

### Универсальные типы артефактов

| Тип | Кому | Пример |
|---|---|---|
| Code | dev, MLE, ops | PR / commit / репа |
| Design | designer | Figma frame / screenshot |
| Document | tech writer, PM, BA | Notion / GDoc / PDF |
| Diagram | SA, BA, архитектор | Miro / Excalidraw |
| Data | analyst, MLE | CSV / dashboard / query |
| Conversation | UX research, product | Запись звонка / транскрипт |
| External | marketing, BD | Cold email / встреча / пост |

### Что обрезано в публичной части

| Поле | Public | Paywall |
|---|---|---|
| Тип артефакта | ✓ | ✓ |
| Категория задачи (backend / UX research / marketing) | ✓ | ✓ |
| Объём (1 PR / 3 Figma frames / 5 interviews) | ✓ | ✓ |
| Время потрачено | ✓ | ✓ |
| Агрегированные skill + behavior scores | ✓ | ✓ |
| Сам артефакт (код, дизайн, документ) | ✗ | ✓ |
| Текст what-I-did + note | ✗ | ✓ |
| Название проекта / бизнес-идея | ✗ (по умолчанию) | ✓ |
| Кто что прокомментировал | ✗ | ✓ |

Команда может выставить toggle «Project name public» — если хайпят, открывают; если боятся утечки идеи — anonymous (публично «Team #42»).

### Daily peer scoring — правила

- Каждый день каждый оценивает каждого тиммейта (skill + behavior на каждый артефакт)
- **Не оценил 2 дня подряд → автокик из команды**
- Уведомления-напоминалки до этого
- **Менять оценку задним числом нельзя**
- Оценка анонимна для тиммейта, не анонимна для админа и нанимателя

### Пример работы скоринга

Дев пишет: «Сделал 5 PR, всё работает». Тиммейты приходят: «один PR на самом деле сломал прод → skill минус», «в чате на ревью был мудаком → behavior минус», остальное норм.

## Структура спринта

- **Длина:** 1 месяц (или 2 недели на первую волну — снизить барьер входа)
- **Ритм:** ~4 часа в день реальной работы
- **Бюджет команды:** ~$200 на всех (Claude, трафик)

### Playbook лидера (черновой)

1. День 1: знакомство, выбор ниши, договорённость о ритме
2. Дни 2–3: 5 разговоров с потенциальными пользователями
3. Дни 4–7: что строим, роли
4. Недели 2–3: MVP
5. Неделя 4: запуск, первые цифры, итоговый отчёт

### Открытие лога

**Лог открыт сразу, с первого дня.** Не через неделю, не через 3 дня. Builder может в любой момент сам зашарить публичную часть. Это и есть фишка: человека могут нанять посреди спринта — и это нормально.

## Монетизация

- **Builders** — бесплатно. Возможный success fee если наняли (отложено).
- **Employers (компании)** — платный доступ к логам с верификацией. На старте: договариваешься руками, без биллинга.
- **Individuals** (любопытные) — могут открыть полный лог за **$500**. Дорого специально — отсекает праздное любопытство, оставляет рекрутеров. Защита от утечки идей конкурентам идёт бонусом.

## Лендинг — TrustMRR-style

Главная страница не «продай идею + waitlist», а **витрина живых команд**:

- «Вот команды, которые сейчас строят»
- «Вот кусочки их логов — публичная часть»
- «Можешь зайти посмотреть, присоединиться, создать свою»
- «Хочешь нанять — заплати, открою»

**На старте — фейк-данные.** Без стеснения. Аудитория №1 на лендинге — безработные builder'ы, им нужно увидеть, как это выглядит. Постепенно фейк разбавится реальным. Если придёт наниматель и купит, пока пусто — деньги вернуть, объяснить что через 2 недели появится реальное.

Текущая `/cohort` страница архитектурно уже близка к этому — нужно только наполнить и поднять с подстраничного уровня (вероятно, частью главной).

## Открытые вопросы

### 1. Механизм набора в команду

Принципы понятны: низкое трение, большое доверие, вайб > формальные требования, краткий созвон 20-30 мин на финал. Конкретного механизма пока нет.

**Варианты:**

- **A. Tinder через лидера.** Команды-в-формировании висят лентой. Builder жмёт «I'm in» + 2 строки + LinkedIn. Лидер свайпает accept/decline. 48 часов на формирование, иначе команда расформировывается. Требует чтобы лидеры пришли первыми.
- **B. Open lab.** Первые 10–15 builder'ов регятся в общий «лаб», без команд. Первые 3 дня — общая вводная задача (например: «найди 3 ниши, представь 5-мин pitch в общем чате»). За эти 3 дня видно кто с кем сошёлся, кто реально работает. К 4-му дню сами разбиваются на команды по симпатии. Лидер не назначается — выявляется. Решает chicken-and-egg.

**Текущее предложение:** B на первую когорту, A когда лидеров достаточно.

### 2. Дальнейшие детали лога, которых ещё нет

- Как агрегировать ежедневные скоры в видимый «рейтинг»? Среднее по дням? Тренд?
- Что показывать наниматилю как summary — простой score? Distribution? Качественный snippet?
- Как выглядит UI для записи дня — форма, чат-инпут, drag-and-drop артефактов?

### 3. Юридическое и доверие

- Что подписывает builder при входе (NDA с командой? Условия публикации лога?)
- Что подписывает employer при покупке (NDA на содержимое логов? Запрет копирования?)
- IP — кому принадлежат стартапы (предположительно: 100% команде, платформа не претендует, но это надо явно зафиксировать).

### 4. Защита от gaming'а

- Что если команда из 5 друзей договорилась накручивать друг другу скоры?
- Простейший вариант — admin (ты) видит outliers (все 5/5 у всех всегда — подозрительно), вручную влезаешь.
- Долгосрочно — может случайные cross-team peer reviews от builder'ов из других команд?

### 5. Платформа после спринта

- Что происходит с логом, когда спринт закончился? Висит постоянно? Архивируется?
- Может ли builder остаться на платформе и пойти в новый спринт? (Скорее всего да.)
- Может ли builder экспортировать свой лог как «портфолио» (PDF / shareable link)?

## Что есть в коде сейчас (vs концепция)

### Реально работает

- Лендинг (`app/page.tsx`) — длинный, с тёмной неоновой эстетикой. **Описывает старую концепцию: $20/mo, 6 недель, demo day, без work log.** Скелет (Hero / Problem / Idea / How / What / Real talk / Social proof / FAQ) переиспользуется.

- `LandingHeader`, `SubpageHeader`, `Footer` — переиспользуем.

### Мокапы (статичные)

- `/cohort` — 3 захардкоженных проекта (HireMatch AI / TeamPulse / SkillStack) + activity feed с выдуманными SK / MR / LP / DJ. Архитектурно близко к нужному формату витрины, нужно ребрендить под формат лога и наполнить.

- `/dashboard/{board, calendar, activity, docs, playbook}` — статичный shell кабинета.

### Не существует

- Auth, реальный signin
- Любая логика лога, скоринга, набора команд
